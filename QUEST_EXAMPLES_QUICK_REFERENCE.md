## Tab 1: ✓ STRONG EXAMPLES

### Introduction Section:
Shows two real approaches to opening a game review.

**Example #1: Baldur's Gate 3**
```
"Baldur's Gate 3 doesn't just revive a beloved franchise—it redefines 
what a turn-based RPG can achieve when developers treat player freedom 
as a design first principle."

✓ Positions the game
✓ Makes evaluative claim
✓ Defines "real choice" specifically
✓ Shows industry knowledge
```

**Example #2: Outer Wilds**
```
"Outer Wilds is a mystery wrapped in a puzzle wrapped in a spacesim. 
It asks a question—who built this system of planets, and why?—and then 
refuses to show you the answer."

✓ Hooks immediately
✓ Names design philosophy
✓ Contextualizes against trends
✓ Validates the approach
```

---

## Tab 2: ✗ WHAT NOT TO DO

Shows what weak analysis looks like and why it fails.

**Weak Example #1: Generic Praise**
```
❌ "This game is really good. It's fun and has great graphics. 
I loved playing it. If you like games, you should definitely check it out."

Problems:
- Could describe ANY game
- "Fun" and "great" tell us nothing
- No industry context or analysis
- Reads like social media
```

**Weak Example #2: Pure Description**
```
❌ "Elden Ring is an action RPG by FromSoftware. It has an open world 
with dungeons and bosses. You use weapons, magic, and spells."

Problems:
- Reads like Wikipedia
- Just facts, no analysis
- Doesn't explain why this game matters
- Reader doesn't know your perspective
```

**Weak Example #3: Jargon**
```
❌ "The core mechanical paradigms demonstrate a sophisticated synthesis 
of established convention matrices while introducing novel systemic articulations."

Problems:
- Jargon without substance
- Reader has no idea what you mean
- Professional writing is clear, not obscure
- Assumes reader knows the game already
```

**Weak Example #4: Vague Comparison**
```
❌ "This game is kind of like Dark Souls but different. It's pretty good 
and I think people would like it."

Problems:
- "Like Dark Souls but different" tells nothing about what it IS
- "Pretty good" is meaningless
- Hedging suggests uncertainty
- No analysis or evidence
```

---

## Tab 3: EXPLANATION

Explains the principles behind what works and what doesn't.

**Key Principles:**

1. **Position the game in context**
   - Where does it fit? Genre? Franchise? Design tradition?
   - This gives readers a starting point

2. **Make an evaluative claim**
   - Not "this game has X feature" but "this game does X significance"
   - Your opening should argue something

3. **Ground claims in design logic**
   - "This game is good" means nothing
   - "This restricted mechanic system creates precise difficulty curves" explains WHY

4. **Acknowledge the landscape**
   - Show you know what else exists
   - Proves your analysis isn't naive

5. **Set expectations for what follows**
   - By opening with design philosophy, you signal "this reviews systems"

**Formula:**
```
[CONTEXT] + [EVALUATIVE CLAIM] + [WHY IT MATTERS] + [INDUSTRY KNOWLEDGE]
```

## Tab 4: PRACTICE

Guided exercises that compare weak vs. strong answers.

**Exercise 1: What's your core argument?**
```
❌ Weak: "This game is good."
❌ Weak: "This game is fun to play."
❌ Weak: "I really liked this game."

✓ Strong: "This game innovates on formula by..."
✓ Strong: "This game challenges convention by..."
✓ Strong: "This game represents a moment when..."
```

**Exercise 2: Where does it fit?**
```
❌ Weak: "It's a game"
❌ Weak: "Part of a well-known franchise"
❌ Weak: "Came out in 2023"

✓ Strong: "First entry in franchise attempting X approach"
✓ Strong: "Subverts genre's typical Y"
✓ Strong: "Arrives at moment when industry is moving toward Z"
```

## What This Looks Like Across All Sections

### GAMEPLAY Section:

**Strong Example #1: Hollow Knight**
```
"Combat is deliberately restrictive. One attack, one dodge, handful 
of abilities. Against modern action games with combo trees..."

✓ Describes mechanics precisely (not "hard combat")
✓ Contextualizes within industry
✓ Explains why design exists (not just what it is)
✓ Connects systems to player learning
```

**Strong Example #2: Portal 2**
```
"Early puzzles ease you in. Mid-game stacks mechanics. Late-game 
you're solving what seemed impossible at start because your brain 
has been rewired to think in portals."

✓ Shows progression through examples
✓ Analyzes learning curve
✓ Shows HOW the game teaches
```

**Weak Examples: What NOT to Do**
```
❌ "The gameplay is really good"
❌ Lists mechanics like a feature list
❌ "I liked the combat"
❌ "The game is too hard, so it's poorly designed"
```

---

### GRAPHICS Section:

**Strong Example #1: Return of the Obra Dinn**
```
"Every pixel serves the mystery. By committing to 1-bit aesthetic—
the visual language of a 1980s Macintosh—limitation becomes superpower."

✓ Names specific aesthetic and source
✓ Argues form serves function
✓ Addresses potential criticism
✓ Connects visuals to gameplay
```

**Strong Example #2: Control**
```
"Brutalist architecture isn't set dressing—it's the visual language 
of inhuman power. Scale makes players feel small. Cold blues reinforce 
alienation."

✓ Architecture as narrative device
✓ Visual logic connected to theme
✓ Color analyzed functionally
```

**Weak Examples: What NOT to Do**
```
❌ "The graphics are beautiful"
❌ "1920x1080 resolution, dynamic shadows, 4K textures"
❌ "I don't like pixel art"
❌ "Realistic graphics would be better"
```

---

### SOUND Section:

**Strong Example #1: Outer Wilds**
```
"Music doesn't accompany the game—it IS the language. Melodies evolve 
as you learn. First encounter is mysterious. After solving it, same 
theme resolves into clarity. Silence between stars is awe."

✓ Sound analyzed as primary system
✓ Specific musical technique explained
✓ Silence analyzed
✓ Sound connected to player learning
```

**Strong Example #2: DOOM 2016**
```
"Sound design is rhythmic. Weapon shots sync to percussion. Glory kills 
peak at crescendos. Music anticipates what you'll do. This feedback loop 
drives aggression."

✓ Describes sonic approach specifically
✓ Analyzes music-mechanics relationship
✓ Shows how sound reinforces design intent
```

**Weak Examples: What NOT to Do**
```
❌ "The music is really good and makes you feel epic"
❌ "50 tracks, orchestral, composed by John Williams"
❌ "The music got boring so I muted it"
❌ "The audio is fine, there's a soundtrack"
```

---

### VERDICT Section:

**Strong Example #1: A Short Hike**
```
"Game about exploration without pressure, ambition tempered by contentment. 
Seek out if you value games that trust you. Skip if you need constant 
challenge. But if climbing matters more than arriving, this is made for you."

✓ Synthesizes into central thesis
✓ Makes specific recommendations
✓ Contextualizes against convention
✓ Validates unconventional choices
✓ Ends with emotional resonance
```

**Strong Example #2: Elden Ring**
```
"Succeeds because difficulty tells story. Open world creates choice. 
Respects your time by letting you skip or revisit. Not for everyone—
demands patience. But for players who see difficulty as communication, 
it's essential."

✓ Connects all analysis to one thesis
✓ Honest about limitations
✓ Specific about audience
✓ Defends unconventional design
```

**Weak Examples: What NOT to Do**
```
❌ "This is a really good game, I recommend it to everyone"
❌ "I think people might kind of like it, maybe, but I'm not sure"
❌ "Gameplay was good, graphics were pretty, music was nice" (recap, no synthesis)
❌ "Not for serious gamers, skip if you have actual skill" (gatekeeping)
```

## How Students Navigate

2. **Read What NOT to Do** – Recognize mistakes to avoid
3. **Read Explanation** – Understand the principles
4. **Do Practice** – Apply the learning

They can jump to any tab, re-read examples multiple times, and refer back when writing their own work.
