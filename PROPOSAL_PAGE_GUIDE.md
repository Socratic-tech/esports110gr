# Game Proposal Page – Interactive Features Guide

## What Changed?

The proposal page evolved from **static information** to **interactive learning guide**. Students still submit through Canvas, but the page now actively helps them think through their proposal.

---

## Feature Breakdown

### 1️⃣ Expandable Accordions

**Before:** Dense text explaining why proposals matter. Long read.

**After:** Click headers to reveal explanations. Students control the pacing.

```
┌─────────────────────────────────────────┐
│ ▼ Demonstrates Intentionality            │  ← Click to expand
├─────────────────────────────────────────┤
│                                          │
│ You're not reviewing a game at random.   │
│ You've chosen it because...              │
│                                          │
│ Example: "I'm analyzing Elden Ring's...  │
│                                          │
└─────────────────────────────────────────┘
```

**Benefits:**
- Students see headlines before diving into details
- Reduces overwhelming text density
- Supports different reading speeds
- Only one section open at a time (less clutter)

---

### 2️⃣ Example Proposal Tabs

**Before:** One or two static examples embedded in text.

**After:** Three clickable tabs showing different professional approaches.

```
┌─────────────────────────────────────────────────────────┐
│  [Mechanical Deep Dive] [Cultural Context] [Narrative]  │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  Game: Hollow Knight (2017), Indie Metroidvania        │
│                                                          │
│  Justification: Hollow Knight represents a critical...   │
│  [Rest of proposal showing mechanical focus]           │
│                                                          │
│  ✓ Strong Proposal                                       │
└─────────────────────────────────────────────────────────┘
```

**What students see:**
- **Mechanical Deep Dive** – Focusing analysis on game systems
- **Cultural Context** – Analyzing industry/cultural moment
- **Narrative Innovation** – Focusing on storytelling approach

**Benefits:**
- Concrete examples reduce uncertainty ("How do I write this?")
- Different angles show "there's no single right way"
- Real games with real analytical frames
- Can switch between tabs to compare approaches

---

### 3️⃣ Interactive Checklist

**Before:** Written list of required components scattered through text.

**After:** Visual checklist students can check off as they plan.

```
☐ Game Title & Information
  Developer, release year, platform(s), genre

☑ Justification & Relevance (1–2 paragraphs)
  Why this game matters and why it's worthy of analysis
  ← Checked when user decides they understand this part

☐ Historical or Cultural Context (1 paragraph minimum)
  Industry trends, cultural significance, design lineage

☐ Your Analytical Approach (1 paragraph)
  What will you focus on? Mechanics? Narrative? Visuals?

☑ References (minimum 2–3)
  Professional reviews, interviews, academic articles
```

**Visual Feedback:**
- Unchecked items: Light gray background
- Checked items: Blue background (confirming understanding)
- Hover effect: Subtle shadow (interactive affordance)

**Benefits:**
- Keeps requirements visible while thinking
- Provides progress without gamification
- Students self-assess understanding
- Can be toggled on/off as thinking evolves

---

### 4️⃣ Proposal Planner Tool

**Before:** Students had to jump straight to Canvas to start writing.

**After:** Structured thinking tool breaks down the proposal task.

```
PROPOSAL PLANNER
══════════════════════════════════════════════════════════

Game Information
├─ Game Title *
│  └─ Input: [Your Game Here________________]
│     Hint: Include the year and developer
│
├─ Platforms & Genre
│  └─ Input: [Platform and genre_____________]

Why This Game?
├─ What makes this game worthy of analysis? *
│  └─ Text area: [Long-form thinking space]
│     Hint: Don't just say "it's good"...

Context
├─ What's the broader context? *
│  └─ Text area: [When? Industry? Design tradition?]
│     Hint: Show you understand this game's place...

Your Analysis Focus
├─ What will you analyze? *
│  └─ Text area: [Mechanics? Narrative? Art? All?]
│     Hint: Be specific. Don't analyze everything...

References
├─ What sources will you cite? *
│  └─ Text area: [List 2-3 sources you've found]
│     Hint: These prove you've done research

══════════════════════════════════════════════════════════
ℹ️ Note: This helps you think. Submit actual proposal via Canvas.
```

**How Students Use It:**
1. Fill in planner while thinking (5-10 minutes)
2. Have clear outline of their thinking
3. Either copy refined notes to Canvas, or use as springboard
4. Submit actual proposal through Canvas

**Benefits:**
- Breaks overwhelming task into steps
- Reduces blank-page anxiety
- Creates thinking scaffold
- **No saving** means transparent data flow (only they have it)

## Interaction Examples

### Scenario 1: Student Explores Concepts

1. Student reads: "A game proposal does several things"
2. Student sees accordion headers but text is hidden
3. Student clicks "Demonstrates Intentionality"
4. That section expands with explanation + example
5. Student reads and understands
6. Student clicks "Establishes Credibility" to learn next point
7. Previous section collapses (one open at a time)
8. Student proceeds to other sections as needed

**Result:** Student learns at own pace, text doesn't feel overwhelming

### Scenario 2: Student Views Examples

1. Student reads: "Here are real proposal excerpts"
2. Student sees three tabs: Mechanical, Cultural, Narrative
3. Student clicks "Cultural Context" tab
4. That tab becomes visually active (bold text, background)
5. Content smoothly appears showing Control proposal
6. Student reads approach focused on environmental storytelling
7. Student clicks "Mechanical Deep Dive" tab
8. Content switches to Hollow Knight proposal
9. Student sees *different analytical approach* using same format
10. Student realizes there are multiple valid ways to analyze

**Result:** Concrete examples reduce anxiety and show flexibility

### Scenario 3: Student Plans Proposal

1. Student has picked a game (let's say *Outer Wilds*)
2. Student opens Proposal Planner
3. Under "Game Information" types: "Outer Wilds (2019), Mobile Dynamix"
4. Under "Why This Game?" types: "Outer Wilds represents an evolution in how puzzle-based exploration games communicate narrative"
5. Under "Context" explains the 2019 moment for space exploration games
6. Under "Analysis Focus" writes: "I'm analyzing how audio architecture (composer Andrew D. Huang) creates emotional resonance"
7. Under "References" lists: "Polygon review, GDC postmortem, scholarly article on audio narrative"
8. Student has clear outline without writing final prose
9. Student uses this as blueprint for Canvas submission

**Result:** Structured planning prevents rambling, creates clear outline

## Interactive vs. Static Comparison

| Element | Static Version | Interactive Version |
|---------|---|---|
| "Why proposals matter" | Long paragraph | 4 expandable sections with examples |
| Game examples | 1-2 embedded examples | 3 switchable tabs |
| Requirements | Written list | Checkable checklist |
| Planning | Blank Canvas form | Structured planner tool |
| Student agency | Passive reading | Active exploration |
| Cognitive load | High (lots of text) | Low (reveals on demand) |
| Time to understand | 5-10 minutes | 3-5 minutes |
| Anxiety level | High (what do I write?) | Low (here's a structure) |

## Design Principles

### 1. Real Interactivity Only
- ✅ Accordions help students control pacing
- ✅ Example tabs show multiple valid approaches
- ✅ Checklist provides self-assessment
- ✅ Planner scaffolds thinking

- ❌ No fake "progress" tracking
- ❌ No points or badges
- ❌ No gamification mechanics
- ❌ No data sent anywhere (only in browser memory)

### 2. Transparency
- No hidden functionality
- Clear that planner notes don't save
- Obvious that Canvas is submission system
- Explicit: "This tool helps you think"

### 3. Accessibility First
- All interactive elements keyboard accessible
- Clear focus indicators
- High contrast colors
- Semantic HTML

### 4. Pedagogical Intent
- Every feature serves learning
- Accordions = cognitive load management
- Examples = professional modeling
- Checklist = self-assessment
- Planner = scaffolded thinking

## What's Still Static?

These sections remain static (intentionally):

- **Submission Expectations** – Clear, doesn't need interaction
- **Tips for Success** – Reference list, read as needed
- **Next Steps** – Simple call to action to Training Quest

Not everything needs to be interactive. Static content is often better for reference information.

## Accessibility Features

✓ **Keyboard Navigation**
- Tab to navigate between interactive elements
- Enter/Space to activate buttons
- Arrows not needed (linear flow)

✓ **Focus Management**
- Clear focus indicators (blue outline)
- Focus follows logical page order
- No focus traps

✓ **Visual Design**
- Color + shape indicates state (not color alone)
- Sufficient contrast (WCAG AA+)
- Large touch targets (44px minimum)

✓ **Semantic HTML**
- Real button elements (not divs)
- Real form labels (for attributes)
- Proper heading hierarchy

## How to Use This in Your Classroom

### First Week
1. **Share link** with students in Canvas
2. **Demonstrate in class** – Expand an accordion, click a tab, check off items
3. **Assign optional** – "Explore the proposal page before starting your proposal"

### During Proposal Writing
1. Students explore accordions to understand requirements
2. Students view examples to see valid approaches
3. Students use planner to outline thinking
4. Students submit actual proposal through Canvas (with planner as reference)

### Feedback
1. Instructors respond in Canvas with approval/revision
2. Students can iterate using planner tool
3. No interaction with the tool itself is required
